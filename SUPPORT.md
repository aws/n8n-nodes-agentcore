# Support

## Getting help

This project is community-maintained. For bug reports, feature requests, or
usage questions:

- **GitHub Issues:** <https://github.com/aws/n8n-nodes-agentcore/issues>
  (replace with the final repo URL before publish)
- **n8n Community Forum:** <https://community.n8n.io/> — post in the
  community-nodes category with the `n8n-nodes-agentcore` tag

## Support scope

**What this project supports:**

- Bugs in the node's interaction with AgentCore APIs
- Documentation gaps or inaccuracies
- Reasonable feature requests aligned with the roadmap in README.md

**What this project does not support:**

- Issues with Amazon Bedrock AgentCore itself — please use
  [AWS Support](https://aws.amazon.com/support/)
- Issues with n8n itself — please use the
  [n8n community forum](https://community.n8n.io/)
- Issues with third-party MCP servers, tool providers, or foundation models
  invoked through this node — please contact the respective vendors
- Custom workflow debugging or integration consulting

## Response expectations

Maintainers respond to GitHub issues on a best-effort basis. We aim to triage
new issues within five business days. For security issues, please see
[SECURITY.md](./SECURITY.md), which has a formal disclosure process with
defined response SLAs.

## Commercial support

For commercial support of Amazon Bedrock AgentCore, contact
[AWS Support](https://aws.amazon.com/support/).
